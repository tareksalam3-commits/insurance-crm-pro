# سير عملية طلب الانضمام المحسّن

## 📋 نظرة عامة

تم إعادة تصميم عملية طلب الانضمام لتتبع السلسلة الهرمية الصحيحة للشركة:

```
الوكيل (Agent)
    ↓
اختيار الشركة
    ↓
اختيار المراقب العام (General Supervisor)
    ↓
اختيار المراقب (Supervisor)
    ↓
اختيار رئيس المجموعة (Group Leader)
    ↓
إرسال الطلب
    ↓
موافقة المراقب
    ↓
إرسال إيميل إعادة تعيين كلمة المرور
    ↓
الوكيل ينشئ كلمة المرور الخاصة به
```

---

## 🔄 خطوات العملية

### **1. ملء نموذج طلب الانضمام**

الوكيل يملأ البيانات التالية بالترتيب:

| الحقل | النوع | الوصف |
|------|------|-------|
| **الاسم الكامل** | نص | اسم الوكيل الجديد |
| **البريد الإلكتروني** | بريد | البريد الإلكتروني للحساب الجديد |
| **الشركة** | قائمة منسدلة | الشركة التي يريد الانضمام إليها |
| **الوظيفة** | قائمة منسدلة | الدور المطلوب (عادة: وكيل) |
| **المراقب العام** | قائمة منسدلة | يتم تحميلها بناءً على الشركة المختارة |
| **المراقب** | قائمة منسدلة | يتم تحميلها بناءً على المراقب العام المختار |
| **رئيس المجموعة** | قائمة منسدلة | يتم تحميلها بناءً على المراقب المختار |

### **2. التحقق من الصحة**

قبل إرسال الطلب، يتم التحقق من:
- ✅ جميع الحقول المطلوبة مملوءة
- ✅ الشركة محددة
- ✅ المراقب العام محدد
- ✅ المراقب محدد
- ✅ رئيس المجموعة محدد (للوكلاء)

### **3. إرسال الطلب**

عند الضغط على "إرسال طلب الانضمام":
- يتم حفظ الطلب في `registrationRequests` collection
- الحالة: `pending`
- يتم إرسال إشعار للمراقب المختار

### **4. موافقة المراقب**

المراقب يذهب إلى صفحة "طلبات الانضمام" ويرى الطلب:
- يمكنه الموافقة أو الرفض
- عند الموافقة:
  - ✅ يتم إنشاء حساب Firebase Auth
  - ✅ يتم إضافة المستخدم في `users` collection
  - ✅ يتم إضافة الوكيل في `agents` collection
  - ✅ يتم تحديث حالة الطلب إلى `approved`
  - ✅ يتم إرسال إيميل بـ رابط إعادة تعيين كلمة المرور

### **5. إعادة تعيين كلمة المرور**

الوكيل يستقبل إيميل يحتوي على رابط:
- الرابط: `https://insurance-crm-pro-theta.vercel.app/reset-password?oobCode=...`
- يضغط على الرابط
- ينتقل إلى صفحة `/reset-password`
- يدخل كلمة مرور جديدة
- حسابه يصبح جاهزاً للاستخدام

### **6. الدخول للنظام**

الوكيل يدخل بـ:
- البريد الإلكتروني
- كلمة المرور الجديدة

## 🔧 التحسينات المطبقة

### **قبل:**
- ❌ اختيار المدير المباشر فقط
- ❌ لا توجد سلسلة هرمية واضحة
- ❌ الطلب يبقى موجود بعد الموافقة
- ❌ لا توجد رسالة تأكيد

### **بعد:**
- ✅ سلسلة هرمية كاملة (4 مستويات)
- ✅ تحميل ديناميكي للخيارات حسب الاختيار السابق
- ✅ الطلب يختفي بعد الموافقة تلقائياً
- ✅ رسائل تأكيد واضحة
- ✅ معالجة أفضل لأخطاء الإيميل

---

## 📊 الحقول المحفوظة

عند إرسال الطلب، يتم حفظ:

```typescript
{
  id: string;                    // معرّف الطلب
  displayName: string;           // اسم الوكيل
  email: string;                 // البريد الإلكتروني
  companyId: string;             // معرّف الشركة
  companyName: string;           // اسم الشركة
  requestedRole: UserRole;       // الدور المطلوب
  managerId: string;             // معرّف المدير المباشر (رئيس المجموعة)
  managerName: string;           // اسم المدير المباشر
  status: 'pending' | 'approved' | 'rejected';
  createdAt: timestamp;
  approvedAt?: timestamp;
  approvedUid?: string;
}
```

## 🚀 الاستخدام

### **للوكيل الجديد:**
1. اذهب إلى صفحة تسجيل الدخول
2. اختر تبويب "طلب انضمام"
3. ملأ البيانات بالترتيب المطلوب
4. اضغط "إرسال طلب الانضمام"
5. انتظر الموافقة
6. افتح الإيميل واضغط على رابط إعادة تعيين كلمة المرور
7. ادخل كلمة مرور جديدة
8. سجّل الدخول بحسابك الجديد

### **للمراقب:**
1. اذهب إلى صفحة "طلبات الانضمام"
2. اعرض الطلبات المعلقة
3. اضغط "موافقة" أو "رفض"
4. عند الموافقة، سيتم إرسال إيميل للوكيل تلقائياً

## ✅ حالات الاختبار

| الحالة | النتيجة المتوقعة |
|--------|-----------------|
| ملء جميع الحقول بشكل صحيح | ✅ الطلب يُرسل بنجاح |
| ترك حقل مطلوب فارغاً | ❌ رسالة خطأ واضحة |
| اختيار شركة بدون مراقبين عاميين | ⚠️ رسالة تحذير |
| الموافقة على الطلب | ✅ إيميل يُرسل للوكيل |
| الرفض | ✅ الطلب يُرفع من القائمة |

## 📝 ملاحظات

- جميع الاختيارات تُحمّل ديناميكياً بناءً على الاختيارات السابقة
